# 路由決策參考 — pptx vs HTML

Slide Builder 依「交付方式 + 內容性質 + 風格訴求」三軸決定走哪個底層 skill。
`brand.yaml` 的 `output.format` 非 `auto`（明寫 `pptx` / `html`）時直接照辦，跳過判斷。

## 決策樹

```text
使用者要做投影片
├── 對方需要「打開來改」（PowerPoint / Google Slides）？ ── 是 ──→ pptx
├── 要交付一個可上傳/email 的檔案，給不懂技術的人？ ────── 是 ──→ pptx
├── 內容是大量表格 / 財報 / benchmark / 數據圖表？ ──────── 是 ──→ pptx（SVG 圖表）
├── 是培訓課件 / 高資訊密度逐字稿？ ──────────────────── 是 ──→ pptx
├── 要符合企業品牌範本 / 公司簡報規範？ ────────────────── 是 ──→ pptx
│
├── 是線下演講 / 分享會 / demo day / 產品發布會？ ──────── 是 ──→ HTML（guizang）
├── 要強視覺衝擊 / 雜誌感 / 瑞士風 / 強個人風格？ ───────── 是 ──→ HTML（guizang）
├── 想要線上連結、一次做完不用翻頁工具？ ──────────────── 是 ──→ HTML（guizang）
│
└── 都不明確 → 問使用者「這份簡報做完之後，別人主要會：(a) 打開來改 (b) 你站著講」
   (a) → pptx     (b) → HTML
```

## 能力對照矩陣

| 維度 | Anthropic pptx skill | guizang-ppt-skill |
|------|---------------------|-------------------|
| 輸出 | `.pptx`（OOXML）| 單檔 `.html` |
| 可二次編輯 | ✅ PowerPoint/Google Slides | ❌ 靜態，改 code |
| 多人協作 | ✅ | ❌ |
| 數據表格 / 圖表 | ✅ 強（SVG 向量圖表）| ⚠️ 弱（重視覺非重數據）|
| 視覺驚艷度 | ⚠️ 中（受 PPT 引擎限制）| ✅ 高（WebGL / 排版藝術）|
| 風格自訂 | ✅ 任意 brand 色/字 | ❌ 鎖預設主題（雜誌 5 套 / 瑞士 4 套）|
| 離線分享 | ✅ 檔案 | ✅ 單檔可寄 |
| 翻頁體驗 | PPT 播放 | 鍵盤/滾輪/觸控橫向翻頁 |
| 適合 | 交付物、報告、課件、財報 | 演講、發布會、demo、個人分享 |

## brand kit → guizang 預設主題映射

guizang 不吃任意 hex（刻意保護美學）。HTML 路由時，依 `brand.accent` 色相 + `theme_hint` 映射到最接近的預設：

**先決定風格 A / B**（依 `theme_hint`）：

| theme_hint 關鍵字 | 風格 |
|------|------|
| 雜誌風 / 人文 / 故事 / 文化 / Monocle | A · 電子雜誌風 |
| 瑞士風 / 極簡 / 網格 / 數據驅動 / 科技 / 工程 | B · 瑞士國際主義風 |

**再選主題色**（accent 色相就近）：

| 風格 | 預設主題（從中選最接近 brand.accent）|
|------|------|
| A | 墨水經典(通用) / 靛蓝瓷(藍·科技) / 森林墨(綠·自然) / 牛皮紙(暖·人文) / 沙丘(藝術) |
| B | IKB 克萊因藍 / 檸檬黃 / 檸檬綠 / 安全橙（四選一 accent）|

> 映射後務必告知使用者：「網頁 deck 走精選預設色，你的 brand 色作為選色依據，非逐 hex 套用。」

## pptx 路由的 SVG 圖表原則

- 圖表（長條/折線/圓餅/流程）一律用 **SVG 向量**插入，NEVER 貼點陣圖截圖 —— Why: 向量在投影放大不糊、檔案小、省 token
- 色彩取 `brand.primary` / `accent`，資料系列用同色相不同明度，不要七彩
- 一張投影片一個核心圖表 + 一句 takeaway，不要圖表疊圖表
