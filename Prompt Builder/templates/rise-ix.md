# RISE-IX Template — 創作 + 對風格

> 創意寫作、要對某個風格、要 multi-shot 範例對齊時用。

```markdown
<role>
<!-- 創作角色：SaaS 文案寫手 / 技術 blogger / podcast 講稿 -->
</role>

<instructions>
<!-- 整體要做什麼 -->
</instructions>

<steps>
1.
2.
3.
</steps>

<examples>
**參考好範例**：
- <好範例 1>
- <好範例 2>
- <好範例 3>

**避免這種**：
- <壞範例 1>
- <壞範例 2>
</examples>
```

## 範例（landing page hero）

```markdown
<role>SaaS 文案寫手</role>
<instructions>寫 5 個 hero headline 候選，產品是 AI 程式碼審查工具</instructions>
<steps>
1. 抓核心價值（自動化、深度、速度）
2. 各候選用不同 angle（功能 / 結果 / 對比 / 情感 / 數據）
3. 每候選附 1 句副標
</steps>
<examples>
**參考好範例**：
- 「Ship faster. Less stress.」（Linear）
- 「The web framework for production」（Next.js）
- 「Stop writing tests by hand」（明確、有對比）

**避免這種**：
- 「Revolutionary AI-powered solution for modern teams」（空話、贅字）
- 「Boost your productivity 10x」（爛大街）
</examples>
```

## 填寫提示

- **Examples 是 RISE-IX 的核心**：3 個 good + 2 個 bad 是黃金比例。
- **good 範例要多元化**：不要 3 個都是同一家公司的，AI 會學成單一風格。
- **bad 範例 = 反向校準**：明確告訴 AI 哪些是「不要學」的。
- 沒有現成範例 → 改用 RISEN（強制流程）或 CO-STAR（強制風格描述）。
